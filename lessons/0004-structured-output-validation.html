Structured Output Validation

Enforce JSON schema layout guarantees on your fine-tuned adapters using guided decoding.

1 Why Guarantee JSON?

In medical NLP applications, extraction outputs are fed directly into Electronic Health Record (EHR) databases, visualization frontends, or diagnostic APIs. A single missing double quote, extra field, or conversational preamble (e.g. "Sure, here is the JSON extraction...") will crash downstream parsers.

We cannot rely purely on prompt engineering to get perfect JSON. We need programmatic constraints during text generation.

2 Two Paradigms: Parsing vs. Guided Decoding

There are two primary approaches to enforcing outputs:

  • Post-Hoc Parsing & Retrying (e.g. Instructor): The model generates standard free-form text. A parser (like Pydantic) checks the output. If it fails, the script queries the model again with the validation error. This wastes tokens and VRAM.
  • Guided Decoding (e.g. Outlines, Guidance): Modifies the model's token sampling probabilities at each generation step. Tokens that would violate the target JSON schema or Regex are assigned a probability of zero. The model is **mathematically guaranteed** to generate a valid schema representation on the first attempt.

4 Code: Guided Inference with your QLoRA Adapter

Create a script named scripts/structured_inference.py. This script loads your fine-tuned model and adapter from Lesson 3, defines a Pydantic schema, and guides token generation to produce guaranteed JSON:

import torch
from pydantic import BaseModel, Field
from peft import PeftModel
from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig
import outlines

# 1. Define the Pydantic schema matching your clinical database requirements
class ClinicalAnamnesis(BaseModel):
    chief_complaint: str = Field(description="The primary clinical complaint or symptom")
    duration: str = Field(description="The duration of the complaint")
    medications: str = Field(description="List of current medications or 'none'")

# 2. Configs
model_id = "NousResearch/Meta-Llama-3-8B-Instruct"
adapter_id = "./qlora_output" # Path to adapter weights saved in Lesson 3

bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_use_double_quant=True,
    bnb_4bit_compute_dtype=torch.bfloat16
)

print("=== 1. Loading Base Model ===")
base_model = AutoModelForCausalLM.from_pretrained(
    model_id,
    quantization_config=bnb_config,
    device_map="auto"
)
tokenizer = AutoTokenizer.from_pretrained(model_id)

print("=== 2. Attaching Fine-Tuned QLoRA Adapter ===")
model = PeftModel.from_pretrained(base_model, adapter_id)
model.eval() # Set model to evaluation mode

print("=== 3. Wrapping Model with Outlines ===")
# Wrap the PyTorch model and tokenizer with Outlines
outlines_model = outlines.models.Transformers(model, tokenizer)

# Compile a JSON generator bound to the Pydantic schema
generator = outlines.generate.json(outlines_model, ClinicalAnamnesis)

# 4. Define raw clinical anamnesis query
new_clinical_note = "Patient complains of sharp lower back pain since yesterday morning. Took Acetaminophen 500mg once."

prompt = (
    f"<|begin_of_text|><|start_header_id|>system<|end_header_id|>\n\n"
    f"You are a clinical NLP assistant. Extract structured JSON containing: 'chief_complaint', 'duration', and 'medications'.<|eot_id|>"
    f"<|start_header_id|>user<|end_header_id|>\n\n{new_clinical_note}<|eot_id|>"
    f"<|start_header_id|>assistant<|end_header_id|>\n\n"
)

print("=== 4. Executing Guided Generation ===")
# This is guaranteed to conform to the ClinicalAnamnesis schema structure
structured_data = generator(prompt)

print("\n🎉 Output Successfully Generated and Validated!")
print("===========================================")
print("Type:", type(structured_data))
print("JSON Object:", structured_data)
print("Chief Complaint:", structured_data.chief_complaint)
print("Duration:", structured_data.duration)
print("Medications:", structured_data.medications)

Run the script:

python scripts/structured_inference.py
