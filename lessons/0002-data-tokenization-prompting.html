Tokenization & Prompting for Clinical Text

Understand tokenization mechanics, chat templates, and formatting medical notes for training.

1 What is Tokenization?

LLMs do not read raw strings. Instead, they process text as a sequence of numbers. A tokenizer is responsible for breaking strings into smaller pieces called tokens (which can be whole words, subwords, or individual characters) and mapping them to integer IDs in a predefined vocabulary.

In fine-tuning, you must use the exact same tokenizer that the base model was trained on. A mismatch in vocabulary or special tokens will corrupt the model's understanding.

2 Llama 3 Chat Templates & Special Tokens

To perform instruction tuning (like converting a raw doctor note into structured JSON), we structure our training data using a **chat template**. A template segments inputs into distinct roles (system, user, and assistant) using **special tokens** that mark bounds:

  • <|begin_of_text|>: Starts the token sequence.
  • <|start_header_id|>system<|end_header_id|>: Identifies the system prompt.
  • <|start_header_id|>user<|end_header_id|>: Identifies user query.
  • <|start_header_id|>assistant<|end_header_id|>: Marks the model's response.
  • <|eot_id|>: End of turn identifier.

Hugging Face's apply_chat_template() automates this formatting so you don't need to manually inject these tokens.

3 Practice Code: Tokenizing Clinical Anamnesis

Let's write a Python script using transformers to tokenize a sample clinical note. Make sure you activate your Conda environment (conda activate fine-tuning) and install transformers first:

pip install transformers huggingface_hub

Create a script named scripts/tokenize_anamnesis.py:

from transformers import AutoTokenizer

# We will use the Llama-3 8B Instruct tokenizer
model_id = "meta-llama/Meta-Llama-3-8B-Instruct"
# Note: You can use a local or open equivalent if HF access is restricted, e.g., "NousResearch/Meta-Llama-3-8B-Instruct"
tokenizer = AutoTokenizer.from_pretrained("NousResearch/Meta-Llama-3-8B-Instruct")

# Define the chat dialogue for a clinical notes task
messages = [
    {
        "role": "system",
        "content": "You are a clinical NLP assistant. Extract structured JSON containing: 'chief_complaint', 'duration', and 'medications'."
    },
    {
        "role": "user",
        "content": "Patient presents with severe throbbing headache for 3 days. Denies fever. Currently taking Ibuprofen 400mg as needed."
    }
]

# Apply the model's chat template
formatted_prompt = tokenizer.apply_chat_template(messages, tokenize=False, add_generation_prompt=True)
print("=== Formatted Chat Prompt ===")
print(formatted_prompt)

# Convert prompt string to token IDs
token_ids = tokenizer.encode(formatted_prompt)
print("\n=== Token IDs ===")
print(token_ids[:30], "... (total tokens:", len(token_ids), ")")

# Decode back to verify the tokens
decoded_tokens = [tokenizer.decode([tid]) for tid in token_ids[:15]]
print("\n=== Individual Decoded Subwords ===")
print(decoded_tokens)

Run the script:

python scripts/tokenize_anamnesis.py
